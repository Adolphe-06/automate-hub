1. Quelle configuration de VPS choisir pour n8n ?

Pour faire tourner n8n sans limites, vous devez louer un serveur privé virtuel (VPS). Le choix des ressources dépend de ce que vous allez brancher sur vos workflows :

Utilisation Configuration Recommandée Espace Stockage
Débutant / Petits Workflows 1 vCPU / 2 Go RAM 20 Go SSD
Pro / Agents IA & Workflows Parallèles 2 vCPU / 4 Go RAM 40 Go SSD

Si vous prévoyez d'utiliser des agents IA, plusieurs workflows en parallèle ou des traitements plus lourds, partez directement sur l'option 2 vCPU et 4 Go de RAM pour éviter les crashs de mémoire en pleine exécution.

2. Installation de Docker et déploiement de n8n

Une fois connecté à votre VPS en SSH via votre terminal, nous allons mettre à jour le système et installer Docker en quelques lignes de commande.

Étape A : Mise à jour et installation de Docker

Exécutez les commandes suivantes pour préparer votre environnement Linux :

sudo apt update && sudo apt upgrade -y
sudo apt install docker.io docker-compose -y
sudo systemctl enable --now docker

Étape B : Configuration du fichier Docker Compose

Pour lancer n8n de manière professionnelle et sécurisée, nous allons utiliser un fichier docker-compose.yml. Cela permet de garantir que vos données restent sauvegardées même si le conteneur redémarre.

Créez et ouvrez le fichier :

nano docker-compose.yml

Collez-y la configuration standardisée suivante :

version: '3.8'

services:
  n8n:
    image: docker.n8n.io/n8nio/n8n:latest
    restart: always
    ports:
      - "5678:5678"
    environment:
      - N8N_BASIC_AUTH_ACTIVE=true
      - GENERIC_TIMEZONE=Europe/Paris
    volumes:
      - n8n_data:/home/node/.n8n

volumes:
  n8n_data:

Étape C : Lancement de l'instance n8n

Démarrez votre serveur d'automatisation en arrière-plan avec la commande :

docker-compose up -d
